# KakaoTalk Scheduler

React + TypeScript + Vite 기반의 카카오톡 예약 메시지 도구입니다. 사용자 액세스 토큰을 이용해 **나에게 보내기** 기본 텍스트 메시지를 원하는 시간에 자동으로 전송합니다.

## 주요 기능

- **액세스 토큰 관리**: Kakao Developers에서 발급받은 사용자 액세스 토큰을 입력하고 로컬 스토리지에 보관
- **예약 메시지 생성**: 날짜/시간/메시지 내용을 지정해 예약
- **자동 전송 워커**: 브라우저가 열려 있는 동안 3초 간격으로 예약 메시지 실행
- **실패 재시도 및 수동 전송**: 실패한 메시지는 즉시 재시도 예약, 버튼으로 강제 전송 가능
- **사용 가이드 제공**: 토큰 발급 및 운영 시 유의사항 안내

> ⚠️ 이 프로젝트는 브라우저에서만 동작합니다. 창이 닫히거나 절전 모드가 되면 예약이 실행되지 않습니다. 실서비스에는 백엔드 워커나 서버리스 함수가 필요합니다.

## 실행 방법

```bash
# 의존성 설치
npm install

# 개발 서버 실행 (http://localhost:5173)
npm run dev

# 프로덕션 빌드
npm run build
```

## 카카오 액세스 토큰 발급 요약

1. [Kakao Developers](https://developers.kakao.com)에서 애플리케이션 생성
2. **카카오 로그인** 및 **카카오톡 메시지 → 나에게 보내기** 권한 활성화
3. OAuth 인증을 통해 사용자 액세스 토큰 발급 (유효기간 기본 6시간)
4. 이 앱의 토큰 입력 란에 액세스 토큰을 붙여넣고 테스트 메시지로 검증

자세한 절차는 Kakao Developers 공식 문서를 참고하세요.

## 프로젝트 구조

- `src/App.tsx` : 예약 로직 및 전체 UI
- `src/App.css` : 레이아웃 및 컴포넌트 스타일
- `src/index.css` : 글로벌 스타일 (배경 및 기본 폰트)

## 한계 및 확장 아이디어

- 메시지는 카카오 정책상 **나에게 보내기** 템플릿으로 제한됩니다.
- 백엔드가 없어 창이 닫히면 예약이 멈춥니다.
- 확장하려면 Supabase Edge Functions, Cloud Functions 등으로 워커를 구현해 서버에서 예약을 실행하고, DB에 예약 내역을 저장하는 방식을 권장합니다.
